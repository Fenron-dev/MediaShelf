# Repository Working Agreement

## GitHub Issues

GitHub Issues are the source of truth for all actionable bugs, tasks, features,
improvements, technical debt, and user verification work in this repository.

- At the start of relevant work, review open issues and search open and closed
  issues for duplicates.
- Create or update an issue as soon as a new actionable item is discovered.
- Give each issue a clear scope, acceptance checklist, and links to related
  issues, commits, or pull requests.
- Add a user-verification checklist whenever the user needs to test behavior.
- Keep an issue open while required user-verification items remain unchecked.
- When implementation is complete, record the verification result in the issue,
  check completed criteria, and close it only when no required work remains.
- If only part of an issue is completed, keep it open or create and link a
  follow-up issue for the remaining scope.
- Do not use local TODO lists as a substitute for GitHub Issues.

See `docs/ISSUE_WORKFLOW.md` for the project workflow and issue structure.
